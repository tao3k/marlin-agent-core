** Purpose
This plan turns [[file:40.120-project-scoped-agent-graph-runtime.org][40.120 Project-scoped Agent Graph Runtime]] into code without blocking on [[file:40.130-sandbox-backend-capability-protocol.org][40.130 Sandbox Backend Capability Protocol]].

Architecture calibration: the implementation is Org-first. Org is the core
reference substrate for project memory, evidence, contracts, source spans, and
compact context packs; code slices should be selected because they strengthen
that substrate, not because they are the smallest local patch. See
[[file:../superpowers/specs/2026-06-15-org-first-project-memory-runtime-design.md][Org-first Project Memory Runtime Design]].

Sandbox backend capability remains important, but it is not a prerequisite for the first project-scoped runtime. The first runtime slice is a read-model and protocol slice:

#+begin_example
Project/session/content/tool/memory facts
  -> typed protocol contracts
  -> Org-backed projection
  -> scoped graph query
  -> debug CLI readout
  -> harness replay evidence
#+end_example

No P0 runtime step executes untrusted code through a new sandbox backend. Sandbox execution adapters start only after the read runtime can produce typed scope, query, and receipt evidence.

** Current Anchors
ASP discovery found these existing anchors:

- =crates/marlin-agent-protocol/src/runtime_environment.rs= owns runtime workspace project identity and the current small =RuntimeSandboxPolicy=.
- =crates/marlin-agent-runtime/src/model_route/session.rs= owns child session id derivation and visibility projection for context forks.
- =crates/marlin-org-store/src/memory.rs= owns =MemoryOrgSourceStore=.
- =crates/marlin-agent-core/tests/unit/runtime/facade/debug_cli/graph/query.rs= already tests debug graph query over loop/run receipt facts.
- =crates/marlin-agent-harness/src/runtime.rs= owns runtime/environment visibility evidence and working-copy isolation visibility evidence.

The implementation should extend these boundaries rather than creating a broad new runtime crate.

** Deferred Scope
Defer these until after the runtime read model is stable:

- Nono, CubeSandbox, native OS, or external sandbox execution adapters.
- backend capability selection during actual command execution.
- DataFusion/Arrow query backend.
- live LLM integration for memory promotion or content compression.

The first implementation may define receipt contracts for these future features when needed, but it should not execute them.

=OrgProjectRootCandidate= before running the matching store query entrypoint.

** Slice G: Project Topology Overview Contract
Owner: =marlin-agent-protocol=, =marlin-org-store=, =marlin-org-workspace=,
and =marlin-org-memory=. Status: P2 store-backed read model and debug CLI root
implemented on 2026-06-15 for =agent.topology.v1=,
=GraphQueryFamily::Topology=, topology visibility, compact topology graph
query, typed topology root discovery, and CLI store-backed query execution.

Topology is the project/session/content navigation layer. It records coarse
overview nodes and edges such as project import, root session creation, child
session branch, content-anchor fork, context-pack compression, memory promotion,
and sub-agent route. It must not duplicate raw transcripts, memory claims, or
whole Org shards; it carries source anchors into the detailed Org fact layer.

Implemented:
- =agent.topology.v1= is a standard agent contract with a concrete example;
- =GraphQueryFamily::Topology= and =GraphQueryVisibleSurface::Topology= are
  protocol-visible;
- =MemoryOrgWorkspace::query_topology_graph= projects compact topology overview
  matches from loaded Org nodes;
- =MemoryOrgWorkspace::query_topology_graph_from_store= executes topology graph
  requests over =MemoryOrgSourceStore= roots discovered as =Topology= roots;
- debug CLI graph query exposes =--org-topology-store-root= plus
  =--org-topology-root= as a topology-only read lane, separate from memory and
  tool capability roots;
- topology nodes must carry =TOPOLOGY_ID=, =PROJECT_ID=, =TOPOLOGY_NODE_KIND=,
  =TOPOLOGY_EDGE_KIND=, and =SOURCE_ANCHOR= before they enter the read model.

Tests:
- standard contract library exposes =agent.topology.v1= with contract ids and
  assertion ids;
- standard contract examples validate the topology shape;
- memory workspace standard contract status includes the topology assertions
  without forcing a topology validation receipt on a memory-only document.
- protocol JSON coverage proves topology family serialization and default
  topology surface visibility;
- Org memory coverage proves same-project topology query and external-project
  denial;
- Org store discovery coverage proves =.marlin/topology/project.org= is a typed
  =Topology= project root candidate;
- debug CLI coverage proves store-backed topology query execution, missing-root
  denial, and option conflict rejection for mixed topology/tool lanes.

Next:
1. Wire imported project/session topology writers to emit real topology shards
   under the discovered root layout.
2. Keep topology readout compact: node id, edge kind, source anchor, visibility
   boundary, and relationship reason. Detailed Org facts remain behind their
   source anchors.
