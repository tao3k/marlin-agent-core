** Purpose
This plan turns [[file:40.120-project-scoped-agent-graph-runtime.org][40.120 Project-scoped Agent Graph Runtime]] into code without blocking on [[file:40.130-sandbox-backend-capability-protocol.org][40.130 Sandbox Backend Capability Protocol]].

Architecture calibration: the implementation is Org-first. Org is the core
reference substrate for project memory, evidence, contracts, source spans, and
compact context packs; code slices should be selected because they strengthen
that substrate, not because they are the smallest local patch. See
[[file:../superpowers/specs/2026-06-15-org-first-project-memory-runtime-design.md][Org-first Project Memory Runtime Design]].

Sandbox backend capability remains important, but it is not a prerequisite for the first project-scoped runtime. The first runtime slice is a read-model and protocol slice:

#+begin_example
Project/session/content/tool/memory facts
  -> typed protocol contracts
  -> Org-backed projection
  -> scoped graph query
  -> debug CLI readout
  -> harness replay evidence
#+end_example

No P0 runtime step executes untrusted code through a new sandbox backend. Sandbox execution adapters start only after the read runtime can produce typed scope, query, and receipt evidence.

** Current Anchors
ASP discovery found these existing anchors:

- =crates/marlin-agent-protocol/src/runtime_environment.rs= owns runtime workspace project identity and the current small =RuntimeSandboxPolicy=.
- =crates/marlin-agent-runtime/src/model_route/session.rs= owns child session id derivation and visibility projection for context forks.
- =crates/marlin-org-store/src/memory.rs= owns =MemoryOrgSourceStore=.
- =crates/marlin-agent-core/tests/unit/runtime/facade/debug_cli/graph/query.rs= already tests debug graph query over loop/run receipt facts.
- =crates/marlin-agent-harness/src/runtime.rs= owns runtime/environment visibility evidence and working-copy isolation visibility evidence.

The implementation should extend these boundaries rather than creating a broad new runtime crate.

** Deferred Scope
Defer these until after the runtime read model is stable:

- Nono, CubeSandbox, native OS, or external sandbox execution adapters.
- backend capability selection during actual command execution.
- DataFusion/Arrow query backend.
- live LLM integration for memory promotion or content compression.

The first implementation may define receipt contracts for these future features when needed, but it should not execute them.

** Slice D: Scoped Graph Query Read Model
Owners: =marlin-agent-protocol=, =marlin-org-memory=, =marlin-agent-core=.
Status: Partially implemented on 2026-06-15 for the first debug CLI receipt reader and explicit-file Org memory fixture execution path. Store-backed shard discovery, source-span projection, and tool/session/content families remain pending.

Implement read-only query families:

#+begin_example
org
memory
tool
session
content
#+end_example

The first backend can be in-memory over Org facts. The output must include:

#+begin_example
typed facts
source spans
relationship strength
match reason
visibility decision
expected receipt contracts
#+end_example

Do not return whole documents or raw grep-style output.

Tests:

- debug CLI graph query can return a compact project memory match;
- graph query refuses sibling transcript without explicit link;
- tool capability query returns project-local card before global fallback;
- query output includes source span and reason.

Implementation note: =marlin graph query= now accepts typed =GraphQueryResponse= JSON and returns a compact =ProjectRuntimeQuerySummary= with receipt id, query family, query text, match count, source project ids, source root/session ids, memory/content ids, relationship facts, and score basis points. For debug fixtures, it can execute a local memory query with =--input <GraphQueryRequest.json> --org-memory-fixture <memory.org> [--org-memory-fixture <memory.org> ...] [--receipt-id <id>]=. That path loads explicit Org memory fixture files into =MemoryOrgWorkspace=, runs =query_project_memory_graph=, and returns the same compact summary without exposing whole documents. This proves multiple producer sessions can project into one graph read, but it is not the production recall API. Production recall now starts from =ProjectMemoryRecallRequest=, discovers typed roots through =marlin-org-store=, builds a contract-indexed frontier in =marlin-org-memory=, then returns a bounded =ProjectMemoryContextPack=. Each returned =ProjectMemoryContextFact= now carries the matched Org source span and stable =MEMORY_ID= evidence handle when the source node provides them.

Rust harness verification policy is crate-role aware. The build-support policy
classifies packages as agent runtime, agent harness, protocol contract, Org
workspace, Gerbil/native runtime, native build support, Rust harness policy,
Git/environment boundary, or general library. Each role injects different
=RustOwnerResponsibility= sets and owner-local profile hints while still
requiring performance and stability verification tasks plus report obligations.
This keeps agent-facing verification evidence specific to the crate's actual
responsibility instead of treating every crate as a generic =src/lib.rs= unit.
The policy now emits a dedicated =verification_policy.json= build evidence
artifact. That receipt exposes the package name, crate role, owner profiles,
responsibility labels, role-specific task kinds, performance/stability task
counts, and report-obligation booleans. Agents should read this artifact before
choosing crate-local verification policy because it explains why a crate needs
runtime, protocol, Org persistence, Gerbil/native, Git/environment, or
general-library evidence.
The quality findings receipt also includes =verification_policy.json= in its
agent evidence set, so repair guidance points agents at the role policy before
they edit crate-local verification gates.
Reflection now feeds an explicit =improvement_queue.json= artifact. That queue
is derived from =quality_findings.json= plus =verification_policy.json= and
converts hard errors and warnings into prioritized improvement items with owner,
crate role, repair objective, next action, verification command, and source
evidence. Advice-only healthy reflection stays a healthy empty queue, so agents
do not treat passive evidence as work.

** Done Criteria
P0 runtime is done when:

- graph query requires =GraphQueryContext=;
- project memory is strong across worktrees;
- sibling root transcripts remain hidden by default;
- child sessions can fork from content anchors with bounded visibility;
- project tool capability cards are queryable;
- all output carries source spans and match reasons;
- replay evidence proves the read model without live LLM or sandbox execution.
