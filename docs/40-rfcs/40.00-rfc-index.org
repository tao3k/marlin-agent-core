** Progress Rules

- RFC status values are =Active=, =Closeout-ready=, =Governance=, or =Accepted=.

- A RFC becomes =Closeout-ready= only when implementation evidence, a focused
  validation gate, and verification commands exist in repository-local tests or
  documented release gates.

- A RFC becomes =Accepted= only after its open questions are either resolved or
  explicitly moved to a follow-up RFC or decision document.

- New code work should update this index in the same pass as the implementation
  receipt, so progress remains visible from one file.
