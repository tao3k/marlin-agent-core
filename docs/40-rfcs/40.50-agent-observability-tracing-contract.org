* 40.50 Agent Observability Tracing Contract
:PROPERTIES:
:ID: docs-40-50-agent-observability-tracing-contract
:KIND: rfc
:STATUS: accepted
:END:

** Problem

Marlin is an agent-core workspace. It needs runtime observability for graph-loop execution, provider calls, tool calls, hook dispatch, sub-agent delegation, cancellation, and task failures without becoming a full application platform.

When many agents run concurrently, ordinary string logs are not enough. Operators and tests need stable trace spans and fields that can answer which agent node ran, which executor or hook was involved, which runtime boundary spawned work, and where failures or cancellations occurred.

** Goals

The tracing contract has four goals:

Expose stable spans for agent-core runtime boundaries.

Keep trace fields low-cardinality and safe to aggregate.

Allow local debugging through any =tracing= subscriber without requiring OpenTelemetry or a server process.

Preserve protocol ownership: tracing observes runtime behavior, while typed receipts and evidence remain protocol-owned data.

** Non-goals

This RFC does not introduce a metrics backend, distributed tracing collector, web UI, app server, model provider implementation, or tool catalog.

This RFC does not make =tracing-subscriber= a normal library dependency. Subscribers belong to binaries, harnesses, tests, or later integration crates.

This RFC does not put raw prompts, model responses, tool arguments, or filesystem contents into trace fields. Such payloads must stay in typed receipts, redacted summaries, or explicit debug artifacts.

** Span Taxonomy

The first tracing surface is intentionally small:

| Span | Owner | Meaning |
|------+-------+---------|
| =runtime.task= | =marlin-agent-runtime= | generic runtime-spawned work |
| =runtime.provider= | =marlin-agent-runtime= | provider runtime boundary |
| =runtime.tool= | =marlin-agent-runtime= | tool runtime boundary |
| =runtime.sub_agent= | =marlin-agent-runtime= | delegated sub-agent runtime boundary |
| =runtime.hook= | =marlin-agent-runtime= | hook runtime boundary |
| =agent.provider= | =marlin-agent-kernel= | graph node executed through a provider adapter |
| =agent.tool= | =marlin-agent-kernel= | graph node executed through a tool adapter |
| =agent.sub_agent= | =marlin-agent-kernel= | graph node executed through a sub-agent adapter |
| =hook.dispatch= | =marlin-agent-hooks= | one hook event dispatched through the registry |
| =hook.run= | =marlin-agent-hooks= | one registered hook handler run |
| =harness.execution= | =marlin-agent-harness= | one scenario execution report boundary |
| =harness.result= | =marlin-agent-harness= | completed scenario execution summary |

** Stable Fields

Core fields should stay stable across subscribers:

| Field | Meaning |
|-------+---------|
| =runtime_kind= | provider, tool, sub_agent, hook, blocking, cancellable, or generic |
| =node_kind= | provider, tool, or sub_agent graph node adapter |
| =node_id= | graph node identifier |
| =executor= | graph executor name |
| =hook_event= | hook event name |
| =hook_id= | registered hook identifier |
| =hook_mode= | sync or async hook execution mode |
| =hook_count= | number of matching hook registrations |
| =error= | bounded error string for runtime join failures |
| =scenario_id= | harness scenario identifier |
| =run_id= | graph-loop run identifier |
| =parent_run_id= | parent graph-loop run identifier for delegated sub-agent work |
| =child_run_id= | child graph-loop run identifier when known at span creation time |
| =graph_id= | graph identifier |
| =agent_reference= | stable sub-agent reference such as executor name or configured alias |
| =sub_agent_source= | low-cardinality source classifier for sub-agent delegation |
| =status= | graph-loop execution status |
| =duration_ms= | elapsed harness execution milliseconds |
| =diagnostic_count= | number of execution diagnostics |
| =event_count= | number of runtime events captured in the report |
| =span_count= | number of trace spans captured in the report |

Additional fields may be added when they describe core agent behavior. High-cardinality payloads require explicit redaction or summarization before they enter tracing.

** Protocol Surface

=marlin-agent-protocol= owns the typed observability identifiers that tests and downstream crates should depend on:

| Type | Meaning |
|------+---------|
| =AgentEventTopic= | stable runtime event topic identifier |
| =AgentSpanName= | stable trace span name identifier |
| =AgentTraceSpanRecord= | captured trace span record with span name and low-cardinality fields |
| =AgentExecutionTrace= | compact run-level trace package with status, events, spans, and diagnostics |
| =AgentExecutionTraceSummary= | count-only trace summary for scanning many runs |

Scenario contracts may assert observability through =AgentScenarioStep.expected_event_topics= and =AgentScenarioStep.expected_span_names=. These fields are typed collections, not ad hoc strings.

=marlin-agent-harness= captures runtime events and trace spans into =HarnessExecutionReport=. The report exposes =events=, =trace_spans=, =span_names=, =summary=, =evidence=, =assertion=, and =execution_trace()= so tests can validate both runtime behavior and observability evidence at the same boundary.

=AgentExecutionTrace= is the protocol-owned trace package for one graph run. It exposes span query helpers such as =has_span=, =count_span=, =spans_by_name=, =spans_with_field=, and =find_span_with_field= so downstream Agent core users can inspect captured trace facts by =run_id=, =graph_id=, =status=, executor, or node fields without depending on the harness report type.

=AgentHarness::evaluate_execution_report= validates expected event topics and expected span names against the captured report. Missing topics or span names become diagnostics rather than subscriber-specific failures.

=marlin-agent-runtime::observability= owns the stable span/topic constants and typed helper constructors. Runtime and kernel callsites may use literal span names in =tracing= macros when required by macro syntax, but those literals must match the contract constants and helpers.

** Ownership

=marlin-agent-runtime= owns runtime task spans and cancellation-related runtime events.

=marlin-agent-kernel= owns graph node spans because it has node IDs, executor names, and adapter kind.

=marlin-agent-hooks= owns hook dispatch and handler-run spans because it has hook registry metadata.

=marlin-agent-protocol= owns reusable observability facts such as span names and captured span records.

=marlin-agent-harness= owns scenario execution and result capture spans because it combines runtime events, trace spans, evidence, and diagnostics for tests.

Provider, tool, and model implementation crates may add child spans later, but they should inherit this contract instead of inventing unrelated names.

** Implementation Status

The implementation adds =tracing= as a workspace dependency and instruments runtime, kernel adapter, hook dispatcher, and harness execution boundaries. It provides a harness-local =TraceRecorder= that captures protocol-owned =AgentTraceSpanRecord= values so scenario tests can capture span names and fields without adopting =tracing-subscriber= or OpenTelemetry as normal dependencies.

=marlin-agent-core= re-exports the protocol observability surface, including =AgentEventTopic=, =AgentSpanName=, and =AgentTraceSpanRecord=, so downstream users can stay on the facade crate while the modular ownership remains explicit.

** Acceptance Decision

Accepted for the current observability tracing contract. The accepted boundary
covers protocol-owned span records, runtime spans, kernel adapter spans, hook
dispatcher spans, harness execution spans, and facade re-exports. Additional
process-lifecycle span checklists are follow-up work only when runtime process
ownership expands.
