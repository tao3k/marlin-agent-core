* 40.160 Agent Graph Topology Protocol
:PROPERTIES:
:STATUS: Active
:OWNER_CRATES: marlin-agent-graph, marlin-org-memory, marlin-agent-protocol, marlin-agent-runtime, marlin-agent-core, marlin-gerbil-scheme
:END:
** Purpose
This RFC establishes =AgentGraph= as the topology layer between single-agent
graph-loop execution and future multi-agent orchestration systems.

The immediate goal is not to implement Symphony. The immediate goal is to
define the typed graph protocol that future Symphony-like orchestration can use
without turning =GraphLoopController=, sessions, Org memory, Gerbil policy,
worktrees, sub-agents, hooks, and evidence into one god controller.
** Layer Boundary
Marlin should keep these layers separate:

| Layer | Owns | Must Not Own |
| Runtime | Tokio, cancellation, process, session, worktree, task receipts | topology, delegation policy |
| Loop | =LoopGraph=, =GraphLoopController=, continuation, retry, repair, next action | teams, scheduling, agent election |
| Knowledge | Org, memory, contracts, evidence, reflection receipts | execution policy, topology mutation |
| Strategy | Gerbil POO, policy, continuation, dynamic hook decisions | runtime storage, process/session ownership |
| Agent Graph | agent nodes, edges, capabilities, delegation, coordination receipts | direct tool execution, raw memory bodies |
| Symphony | decentralized coordination algorithms over =AgentGraph= | core runtime, session, loop contracts |

The critical separation is:

#+begin_example
AgentGraph
  -> GraphLoopController
       -> LoopGraph
            -> Runtime
#+end_example

An =AgentNode= may execute a graph loop through a controller. It must not
execute tools directly.

An =AgentEdge= may describe delegation, review, fanout, fanin, voting, or
handoff. It must not own runtime process state.
** Why Now
The current system already has multiple dimensions that imply topology:

- multiple sessions;
- multiple sub-agents;
- multiple worktrees;
- multiple projects;
- graph-loop controller receipts;
- Org memory graph recall;
- Gerbil policy and continuation logic;
- hook and model routing policy.

Without an =AgentGraph= layer, these concerns will drift into
=GraphLoopController= or =marlin-agent-core=. That would couple topology,
strategy, knowledge, and runtime substrate in the same place.
** Symphony Position
Symphony-style systems are useful as orchestration algorithms, not as the core
Marlin substrate.

Public Symphony material frames the system around decentralized multi-agent
coordination with capability records, dynamic task allocation, and result
voting. Marlin should treat that as future =marlin-symphony= behavior built on
top of =AgentGraph=, not as logic embedded inside =marlin-agent-core=.

External reference:

- https://arxiv.org/abs/2508.20019
** Proposed Crates
*** =marlin-agent-graph=
Owns topology contracts:

- =AgentGraphId=
- =AgentNodeId=
- =AgentEdgeId=
- =AgentRole=
- =AgentCapability=
- =AgentNode=
- =AgentEdge=
- =AgentGraph=
- =AgentDelegation=
- =AgentCoordinationReceipt=
- =AgentElectionReceipt=
- =AgentRoutingReceipt=

This crate should be protocol-first and execution-free.
*** =marlin-symphony= (deferred)
Owns higher-level orchestration algorithms after =AgentGraph= stabilizes:

- dynamic coordinator selection;
- capability ledger strategy;
- team formation;
- agent election;
- weighted result aggregation;
- distributed/decentralized scheduling experiments.

This crate should not exist until =AgentGraph= can represent topology and emit
coordination receipts without requiring Symphony-specific policy.
** Core Types
Initial protocol sketch:

#+begin_src rust
pub struct AgentGraph {
    pub graph_id: AgentGraphId,
    pub nodes: Vec<AgentNode>,
    pub edges: Vec<AgentEdge>,
    pub topology_policy: AgentTopologyPolicy,
}

pub struct AgentNode {
    pub node_id: AgentNodeId,
    pub role: AgentRole,
    pub capabilities: Vec<AgentCapability>,
    pub loop_entry: GraphLoopEntryRef,
    pub memory_scope: Option<OrgMemoryScopeRef>,
    pub policy_scope: Option<GerbilPolicyScopeRef>,
}

pub struct AgentEdge {
    pub edge_id: AgentEdgeId,
    pub from: AgentNodeId,
    pub to: AgentNodeId,
    pub kind: AgentEdgeKind,
    pub condition: AgentEdgeCondition,
}

pub enum AgentEdgeKind {
    Handoff,
    Delegate,
    Review,
    Fanout,
    Fanin,
    Vote,
    Escalate,
}

pub struct AgentCoordinationReceipt {
    pub graph_id: AgentGraphId,
    pub selected_node: AgentNodeId,
    pub selected_edge: Option<AgentEdgeId>,
    pub decision: AgentCoordinationDecision,
    pub evidence: Vec<AgentCoordinationEvidenceRef>,
}
#+end_src
** Relationship To Existing Graphs
=LoopGraph= is a single loop execution graph.

=Org/Memory Graph= is a knowledge graph.

=AgentGraph= is a topology graph over agents. Its nodes can reference
=LoopGraph= entrypoints, memory scopes, capabilities, and policy metadata, but
it does not inline any of those bodies.
** Non-goals
- Do not move session creation into =AgentGraph=.
- Do not move worktree creation into =AgentGraph=.
- Do not execute tools directly from =AgentGraph=.
- Do not store Org memory bodies in =AgentGraph=.
- Do not evaluate Gerbil policy inside =AgentGraph=.
- Do not create =marlin-symphony= until the AgentGraph contract has focused
  tests and receipts.

  execution-free.

handoff, fanout/fanin, reviewer election, graph-loop entry references, and
typed coordination receipts. It also proves Org memory and Gerbil policy
scopes stay typed metadata refs instead of becoming memory queries, policy
programs, tools, commands, sessions, or worktrees.
- =crates/marlin-agent-graph/tests/unit/topology/planning.rs= covers
deterministic, capability-first, and policy-scoped planning before runtime
admission.
- =crates/marlin-agent-graph/tests/unit/topology/planning_rejections.rs=
covers invalid graph and missing-root rejection before runtime admission.
- =build-support/marlin-rust-project-harness-policy= classifies
  =marlin-agent-graph= as =AgentTopology= instead of falling through to a
  generic library role.
- =marlin-agent-runtime= consumes =AgentCoordinationPlan= through
  =admit_agent_coordination_plan=, validates references, and projects the root
  =GraphLoopEntryRef= without selecting agents or executing graph loops.
- =marlin-agent-graph= owns =plan_agent_coordination= so deterministic,
  capability-first, and policy-scoped topology planning produce typed
  =AgentCoordinationPlan= and coordination receipts before runtime admission.
- =marlin-agent-graph= owns =AgentPolicyRoutingReceipt= and
=plan_agent_coordination_with_policy_receipt= as the Rust-side adapter for
=PolicyScoped= routing. A policy-scoped graph without a typed Gerbil policy
receipt is rejected before runtime admission; Gerbil cannot manufacture
runtime requests or handlers through this path.
- =marlin-gerbil-scheme= owns the Rust-side native typed projection boundary
for =AgentPolicyRoutingReceipt=. =GerbilAgentPolicyRoutingProjection= declares
the Scheme type manifest, projection contract, native ABI readiness plan, and
native projection request, then projects Scheme values into typed
=AgentGraphId=, =GerbilPolicyScopeRef=, =AgentNodeId=, =AgentEdgeId=, and
evidence refs before =PolicyScoped= planning can consume them.
- =crates/marlin-gerbil-scheme/gerbil/src/marlin/agent-policy-routing-native-projection.ss=
owns the Gerbil-side POO projection producer for AgentGraph policy routing.
The Scheme module emits the same ABI id, type id, schema id, native symbol, and
POO slots that Rust consumes, while avoiding POO-reserved generic slot names by
projecting =routing_decision=, =routing_evidence=, and =evidence_kind= into
Rust-owned typed fields.
- =crates/marlin-gerbil-scheme/gerbil/src/marlin/agent-policy-routing-native.ss=
owns the Gerbil-side native C export for AgentGraph policy routing. It maps the
native request into the existing POO projection producer and writes only the
native slot projection consumed by Rust. The module is a =gsc:= target in
=gerbil/build.ss= because ordinary =gxtest= interpretation cannot load
=c-declare= forms.
- =crates/marlin-gerbil-scheme/src/agent_policy_routing_native.rs= owns the
Rust-side native C ABI wrapper for AgentGraph policy routing. The ABI passes
borrowed UTF-8 slot projections and evidence refs, never JSON or owned byte
protocol frames, then Rust constructs the typed =GerbilSchemeTypedValue= and
projects it through the existing
=project_gerbil_agent_policy_routing_native_receipt= path into
=AgentPolicyRoutingReceipt=.
- =crates/marlin-gerbil-scheme/tests/unit/agent_policy_routing_native= covers
header/layout stability, native status handling, initializer failure, invalid
UTF-8 rejection, and a fake linked selector smoke from native slot projection
to typed AgentGraph routing receipt.
- =crates/marlin-gerbil-scheme/src/aot_runtime/native= now carries an explicit
native AOT profile boundary. The default profile remains Deck runtime, while
=GerbilDeckRuntimeNativeAotProfile::AgentPolicyRouting= derives the
=agent-policy-routing-native~0= artifact stem, AgentGraph policy-routing C ABI
header, required =marlin_agent_policy_routing_initialize= and
=marlin_agent_policy_routing_select_edges= symbols, and
=marlin.agent.policy-routing.native= readiness ABI. The
=marlin-gerbil-native-aot= CLI exposes the same boundary through
=--profile agent-policy-routing=.
- =crates/marlin-gerbil-scheme/tests/unit/aot_runtime= is split into a
folder-first suite and covers both the Deck default profile and AgentGraph
policy-routing profile without mixing the two symbol contracts.
- =build-support/marlin-gerbil-native-build= is now the shared Gerbil native
build-script support crate. It exposes explicit Deck runtime and AgentGraph
policy-routing link entrypoints, profile-specific archive names, and shared
cc/Gambit discovery without making AgentGraph depend on a Deck-named build
support boundary.
- =crates/marlin-agent-policy-routing-native= is the dedicated AgentGraph
policy-routing linked-native crate. Its default build is a no-link typed Rust
crate; the =linked-native= feature uses Rust-owned native build support to
stage =agent-policy-routing=, consumes the generated
=agent-policy-routing-native~0.scm= artifact, then calls the AgentGraph
policy-routing native build entrypoint and exposes a linked selector only after
the real symbols are available.
- =crates/marlin-agent-policy-routing-native/tests/unit/feature.rs= now runs
the real linked selector under =--features linked-native=. The smoke
initializes the staged Gerbil/Gambit link unit once, calls
=marlin_agent_policy_routing_select_edges=, and projects the result into a
typed =AgentPolicyRoutingReceipt= without JSON or text receipt transport.
- =marlin-agent-runtime= owns =project_agent_graph_planning_receipt= so
planning receipts can become runtime admission receipts without giving the
runtime layer topology selection authority, including malformed planning
receipts that report success without carrying a plan.
- =marlin-agent-protocol= owns =AgentGraphProjectionRequest= as the protocol
bridge from topology planning into runtime projection. The request carries a
typed =AgentGraphId=, an =AgentGraphPlanningReceipt=, and an observation
timestamp; it does not carry a controller, tool call, or execution request.
- =marlin-agent-runtime= owns =project_agent_graph_projection_request= and
=check_agent_graph_execution_readiness=. Readiness is a dry-run receipt: it
proves a root =GraphLoopEntryRef= can be projected, but it does not construct
or execute a =GraphLoopExecutionRequest=. The runtime projection request
boundary rejects unsupported request schema ids before graph admission.
- =marlin-agent-test-support= owns the
=agent_graph_readiness_replay_artifact_fixture= evidence row for
=AgentGraph -> planning -> runtime projection -> readiness= with
=graph_loop_execution=false=, =controller_execution=false=, and
=tool_execution=false=.
- =marlin-org-memory= owns =AgentGraphOrgProjectionRequest= and
=AgentGraphOrgProjectionReceipt= for projecting contract-validated Org
topology metadata into a typed =AgentGraph=. The projection uses explicit
=AGENT_GRAPH_*= properties, emits typed memory and Gerbil policy scope refs,
and rejects unvalidated or invalid topology before runtime admission.
- =marlin-agent-core= re-exports the AgentGraph protocol and runtime
projection boundary through its lean facade, with focused facade contract tests
proving Org projection, planning, protocol request construction, and readiness
projection are reachable without moving topology logic into core.
** Landing Status
1. =marlin-agent-graph= is the dedicated protocol-first topology crate.
2. =marlin-agent-protocol= provides the projection request envelope.
3. =marlin-agent-runtime= consumes projection requests and emits admission plus
readiness receipts without executing graph loops.
4. =marlin-agent-core= exposes the protocol and runtime boundary as a facade
only.
5. Org memory projects contract-validated topology metadata into =AgentGraph=
through typed refs; it does not own runtime execution.
6. =PolicyScoped= planning now requires a typed Gerbil policy routing receipt
before it can produce candidate edges.
7. =marlin-gerbil-scheme= validates the AgentGraph policy routing native ABI
projection before converting Scheme values into Rust =AgentPolicyRoutingReceipt=
data.
8. Gerbil now has a real POO producer module for AgentGraph policy routing
projections, with a focused =gxtest= contract.
9. Rust now has the AgentGraph policy routing native C ABI wrapper plus focused
layout/error/projection tests proving the native slot projection enters the
typed receipt path without text serialization.
10. Gerbil now has a compiled native C export module for AgentGraph policy
routing wired into =gxi build.ss= as =gsc:= and =ssi:= targets.
11. Rust native AOT planning now supports an AgentGraph policy-routing profile
and CLI selection via =--profile agent-policy-routing=.
12. AgentGraph policy-routing now has a dedicated linked-native Rust crate and
the old Deck-only build-support package has been replaced with
=marlin-gerbil-native-build= profile-aware support.
13. Package-owned native AOT staging is wired through =gerbil.pkg= /
=build.ss=: =marlin-agent-policy-routing-native --features linked-native= now
stages =agent-policy-routing-native~0.scm= through Gerbil before Rust builds
the object/archive, so the previous =MissingCompiledRuntime= blocker is closed
without reintroducing Rust-side source discovery.
14. The linked AgentGraph policy-routing ABI smoke now calls the real selector
hot path. The native C ABI module is self-contained for linked execution: the
public Scheme POO projection module remains available for policy-plane tests,
while the linked native ABI writes only the routing decision slot and lets Rust
construct the typed receipt from the request plus native projection.

** Acceptance Criteria
- =GraphLoopController= gains no team, scheduling, election, or delegation
