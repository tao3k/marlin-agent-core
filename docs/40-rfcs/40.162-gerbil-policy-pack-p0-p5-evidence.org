** Purpose
This note records the Gerbil policy-pack evidence used by the AgentGraph P0-P5 closure. It keeps complex policy composition in Scheme while preserving the native Scheme types to Rust types boundary.
** Native Receipt Boundary
The policy-pack projection chain is represented as POO receipts, not JSON:
- =module_evaluation_receipt=
- =policy_projection_receipt=
- =native_projection_payload=
- =budget_receipt=
- =catalog_resolution_receipt=
The direct projection helper exposes receipt family ids, nested receipt values, and owner fields as Scheme values that Rust can later consume through typed native projections.

** Rust Build Script Contract
Rust now owns typed contract facts for the crate-shipped Gerbil package driver through =GerbilRuntimeBuildScriptContract=. The contract records:
- package name =marlin-deck-runtime=
- stages =spec=, =compile=, and =clean=
- dependencies =poo-flow= and =gslph=
- source/coverage roots =src=
- special native sources excluded from generic package make
- worker env precedence =GERBIL_BUILD_CORES=, =CARGO_BUILD_JOBS=, =NUM_JOBS=

The runtime asset test validates those facts against the embedded =build.ss= source. This keeps the build/package boundary Rust-owned without adding a Scheme JSON or text protocol to the native runtime path.
** Native Projection Smoke
The Deck runtime native projection readiness test now validates both exported symbols:
- =marlin_deck_runtime_project_poo_policy=
- =marlin_deck_runtime_project_resolved_loop_policy_pack=

The resolved loop policy pack request is checked against =GERBIL_RESOLVED_LOOP_POLICY_PACK_TYPE_ID= and =GERBIL_RESOLVED_LOOP_POLICY_PACK_SCHEMA_ID=, and the Scheme native projection source must contain the ABI id, both symbols, and both projection contracts.
