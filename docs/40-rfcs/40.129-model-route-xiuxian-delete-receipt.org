* Parity Evidence

Xiuxian chat routing projected HTTP input into route input fields:
=precision_tier=, =privacy_tier=, =latency_budget_ms=, =evidence_profile=, and
=artifact_refs=.

Marlin replaces this with =ChatModelRouteRequest::into_admission_request= in
=crates/marlin-agent-route/src/http.rs=, which projects the same fields into
=ModelRouteAdmissionRequest::chat= and then calls
=admit_model_route_with_resolver=.

Xiuxian artifact routing projected image document extract inputs into:

- =task_kind=: =attachment-extract=
- =modality=: =image=
- =source_kind=: =attachment=
- =precision_tier=: =high=
- =privacy_tier=: =private=
- =latency_budget_ms=: =60000=
- =evidence_profile=: =image-document-markdown=
- =artifact_refs=: =source-sha256:*=, =source-suffix:*=, =backend-profile:*=

Marlin replaces this with
=ModelRouteAdmissionRequest::artifact(...).with_artifact_refs(...)=.  The
protocol test
=model_route_admission_request_projects_artifact_source_intent= locks this
projection.

Marlin also provides =ModelRouteArtifactProjection::image_document_extract= as
the direct typed replacement for xiuxian's local =image_route_input= helper.
Downstream code still owns file existence checks, source hashing, suffix
normalization, and workspace-specific backend profile selection.  The Marlin
helper only turns those already-admitted facts into a
=ModelRouteAdmissionRequest=.

The same projection is re-exported through =marlin-agent-core=, so downstreams
can upgrade through the core facade instead of depending directly on
=marlin-agent-protocol=.
