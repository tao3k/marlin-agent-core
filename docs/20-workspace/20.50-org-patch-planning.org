* 20.50 Org Patch Planning
:PROPERTIES:
:ID: docs-20-50-org-patch-planning
:KIND: protocol
:STATUS: active
:END:

=marlin-org-patch= is the source-aware planning boundary between typed workspace
patches and durable Org text. It consumes =WorkspacePatch= operations together
with parser-owned =OrgSourceSpan= records, then returns an =OrgPatchPlan=.

The plan is intentionally not a writer. It contains byte-range edits that a
caller may apply after validating the current file bytes, and diagnostics for
operations that need more precise parser evidence before they can be made
source-safe.

** Native Objects
- =OrgPatchPlanner= lowers typed patch operations into planned source edits.
- =OrgPatchPlan= carries =OrgTextEdit= values and =OrgPatchDiagnostic= values.
- =OrgTextEdit= names the document, byte range, replacement text, and reason.
- =OrgPatchDiagnostic= records the node, operation, and missing source fact.
- =OrgPatchApplier= applies a diagnostic-free plan to caller-owned in-memory
  document text after validating byte ranges.
- =OrgPatchApplyReport= records applied edit count, diagnostics, and changed
  document before/after hashes.

** Planning Rule
The first planner supports append-only Org edits when a node-level source span is
sufficient. Adding checkboxes, links, and evidence references can anchor at the
end byte of the affected node source span.

Operations that require token-level authority return diagnostics. Examples
include replacing a headline TODO keyword, editing a property drawer entry,
marking an existing checkbox, and emitting a child heading when the parent
heading level is not available in the patch input.

** Boundary Rule
=marlin-org-patch= consumes the Org model and patch receipt types. It does not
parse Org documents itself, does not depend on =orgize= directly, and does not
perform filesystem writes.

The applier is also not a writer. It mutates only the document strings supplied
by the caller, and rejects the whole plan if any edit is out of bounds,
overlapping, missing its target document, or not aligned to a UTF-8 character
boundary. Successful reports include deterministic document hashes so the caller
can connect in-memory application to a later filesystem persistence boundary.

** Gerbil Intent Dry Run

=marlin-org-workflow= consumes Gerbil-emitted =WorkspacePatchIntentSpec=
values through =GerbilWorkspacePatchIntentDryRunner=. This is a workflow receipt
boundary, not a source writer: it requires =dry_run_first=, reports affected
Org nodes, leaves =affected_sources= empty, and returns deterministic dry-run
hash placeholders without reading or writing the workspace.

Memory dispatch operations also remain dry-run only at this boundary. The
receipt records the requested target and a rejection reason so a later workflow
can decide whether memory mutation should be enabled with an explicit policy.

Durable source mutation is a separate request. =GerbilWorkspacePatchIntentCommit=
pairs the Gerbil-emitted intent with a caller-selected document and
=OrgSourceWritePolicy=. =OrgWorkspaceSourceCommitter::commit_gerbil_intent=
first enforces the =dry_run_first= gate, then delegates to the normal
parser-owned source commit workflow. Rejected intents return a source receipt
diagnostic without loading or writing the Org document.
