** Runtime

The local runtime used for the first smoke is Homebrew =gerbil-scheme= 0.18.2:

#+begin_src sh
/opt/homebrew/opt/gerbil-scheme/bin/gxi --version
#+end_src

Expected runtime family:

#+begin_src text
Gerbil v0.18.2 on Gambit v4.9.7
#+end_src

The formula conflicts with Ghostscript on the global =gsc= binary. Keep
Ghostscript linked for normal PATH behavior and use the Gerbil absolute path in
Marlin profiles and tests.

** Rust Adapter Fixture

The first executable adapter shim lives at:

#+begin_src text
crates/marlin-gerbil-scheme/fixtures/gerbil/fixed-loop-graph-adapter.ss
#+end_src

It reads the JSON request from stdin, extracts the request =source.text= field,
and recognizes the first smoke forms:

#+begin_src scheme
(loop-graph gerbil-source-loop
  (node provider ask-model (config role planner retries one))
  (node tool run-tool (config mode execute))
  (edge provider tool success)
  (edge tool provider none))
#+end_src

The shim returns a =LoopGraph= JSON artifact whose =graph_id=, node =config=,
and edge =condition= values are derived from that form. =none=, =null=, and
=#f= edge conditions are emitted as JSON =null=.

The adapter now routes parser output through Gerbil-side modules before
emitting JSON:

- =fixtures/gerbil/marlin/protocol.ss= owns constructors/accessors for
  =LoopGraph=, =LoopNodeSpec=, and =LoopEdgeSpec= plus the
  =GerbilCompileResponse= serializer.
- =fixtures/gerbil/marlin/request.ss= owns the =GerbilCompileRequest= JSON
  reader boundary through =:std/text/json=.
- =fixtures/gerbil/marlin/parser.ss= owns the Gerbil reader-backed smoke DSL
  source-to-=LoopGraph= parser.
- =fixtures/gerbil/marlin/adapter.ss= imports the request/parser/protocol
  modules and owns compile dispatch plus response emission.
- =fixtures/gerbil/fixed-loop-graph-adapter.ss= is a thin launcher that imports
  =:marlin/adapter=.

This validates the real =gxi= process boundary and the first Gerbil-side
source-to-IR step before the compiler grows a full package/build boundary.

The launcher requires =GERBIL_LOADPATH= to point at the fixture root so the
=package: marlin= modules resolve:

#+begin_src sh
GERBIL_LOADPATH=crates/marlin-gerbil-scheme/fixtures/gerbil \
  /opt/homebrew/opt/gerbil-scheme/bin/gxi \
  crates/marlin-gerbil-scheme/fixtures/gerbil/fixed-loop-graph-adapter.ss
#+end_src

Run the Rust boundary test explicitly:

#+begin_src sh
direnv exec . cargo test -p marlin-gerbil-scheme command_compiler_can_call_real_gxi_fixture -- --ignored --nocapture
#+end_src

The test is ignored by default because it requires a local Gerbil installation.
