** Runtime

The local runtime used for the first smoke is Homebrew =gerbil-scheme= 0.18.2:

#+begin_src sh
/opt/homebrew/opt/gerbil-scheme/bin/gxi --version
#+end_src

Expected runtime family:

#+begin_src text
Gerbil v0.18.2 on Gambit v4.9.7
#+end_src

The formula conflicts with Ghostscript on the global =gsc= binary. Keep
Ghostscript linked for normal PATH behavior and use the Gerbil absolute path in
Marlin profiles and tests.

** Rust Adapter Fixture

The first executable adapter shim lives at:

#+begin_src text
crates/marlin-gerbil-scheme/fixtures/gerbil/command-adapter.ss
#+end_src

It reads the JSON request from stdin, extracts the request =source.text= field,
and recognizes the first smoke forms:

#+begin_src scheme
(loop-graph gerbil-source-loop
  (node provider ask-model (config role planner retries one))
  (node tool run-tool (config mode execute))
  (edge provider tool success)
  (edge tool provider none))
#+end_src

The shim returns a =LoopGraph= JSON artifact whose =graph_id=, node =config=,
and edge =condition= values are derived from that form. =none=, =null=, and
=#f= edge conditions are emitted as JSON =null=.

It also recognizes a minimal workspace schema smoke form:

#+begin_src scheme
(workspace-schema workspace-record
  (required ID TITLE)
  (todo TODO DONE))
#+end_src

When =expected= is =WorkspaceSchema=, the adapter emits a =WorkspaceSchema=
JSON artifact with =schema_id=, =required_properties=, and =todo_states=.

The adapter now routes parser output through Gerbil-side modules before
emitting JSON:

- =fixtures/gerbil/marlin/protocol.ss= owns constructors/accessors for
  =LoopGraph=, =LoopNodeSpec=, and =LoopEdgeSpec= plus the
  =GerbilCompileResponse= serializer and the supported artifact-kind registry.
- =fixtures/gerbil/marlin/request.ss= owns the =GerbilCompileRequest= JSON
  reader boundary through =:std/text/json=.
- =fixtures/gerbil/marlin/parser.ss= owns the Gerbil reader-backed smoke DSL
  source-to-=LoopGraph= and source-to-=WorkspaceSchema= parsers.
- =fixtures/gerbil/marlin/adapter.ss= imports the request/parser/protocol
  modules and owns the artifact compiler table, table/registry self-check, and
  response emission.
- =fixtures/gerbil/command-adapter.ss= is a thin launcher that imports
  =:marlin/adapter= and calls =run-marlin-command-adapter=.

This validates the real =gxi= process boundary and the first Gerbil-side
source-to-IR step before the compiler grows a full package/build boundary.

** Next Smoke: WorkspacePatchIntent

The next runtime smoke should add =WorkspacePatchIntent= as a JSON artifact
only. The source form should compile through =protocol.ss=, =parser.ss=, and
=adapter.ss= into =WorkspacePatchIntentSpec= with =dry_run_first= set to true.
The smoke must prove the real =gxi= boundary returns the typed artifact and
that the Gerbil-side supported-kind registry still rejects it until the
serializer, parser, compiler table, and Rust tests are all present.

That smoke must not execute a workspace mutation. Patch validation, dry-run
receipts, and durable persistence remain Rust workflow responsibilities after
the command adapter has decoded and kind-checked the artifact.

** Launcher

The launcher requires =GERBIL_LOADPATH= to point at the fixture root so the
=package: marlin= modules resolve:

#+begin_src sh
GERBIL_LOADPATH=crates/marlin-gerbil-scheme/fixtures/gerbil \
  /opt/homebrew/opt/gerbil-scheme/bin/gxi \
  crates/marlin-gerbil-scheme/fixtures/gerbil/command-adapter.ss
#+end_src

The same adapter module can be invoked directly through Gerbil's module
entrypoint without the launcher file:

#+begin_src sh
GERBIL_LOADPATH=crates/marlin-gerbil-scheme/fixtures/gerbil \
  /opt/homebrew/opt/gerbil-scheme/bin/gxi \
  :marlin/adapter
#+end_src

The Rust boundary tests pass that value through =GerbilCommandSpec.env=, not
through a shell wrapper. They cover the source-file launcher and module entry,
successful =LoopGraph= and =WorkspaceSchema= compilation, plus Gerbil-side
rejection of unsupported =expected= artifact kinds and mismatched source forms.

Run the Rust boundary tests explicitly:

#+begin_src sh
direnv exec . cargo test -p marlin-gerbil-scheme command::real_gxi -- --ignored --nocapture
#+end_src

The tests are ignored by default because they require a local Gerbil installation.
