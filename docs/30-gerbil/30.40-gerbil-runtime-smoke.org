** Runtime

The local runtime used for the first smoke is Homebrew =gerbil-scheme= 0.18.2:

#+begin_src sh
/opt/homebrew/opt/gerbil-scheme/bin/gxi --version
#+end_src

Expected runtime family:

#+begin_src text
Gerbil v0.18.2 on Gambit v4.9.7
#+end_src

The formula conflicts with Ghostscript on the global =gsc= binary. Keep
Ghostscript linked for normal PATH behavior and use the Gerbil absolute path in
Marlin profiles and tests.

** Rust Adapter Fixture

The first executable adapter shim lives at:

#+begin_src text
crates/marlin-gerbil-scheme/fixtures/gerbil/command-adapter.ss
#+end_src

It reads the JSON request from stdin, extracts the request =source.text= field,
and recognizes the first smoke forms:

#+begin_src scheme
(loop-graph gerbil-source-loop
  (node provider ask-model (config role planner retries one))
  (node tool run-tool (config mode execute))
  (edge provider tool success)
  (edge tool provider none))
#+end_src

The shim returns a =LoopGraph= JSON artifact whose =graph_id=, node =config=,
and edge =condition= values are derived from that form. =none=, =null=, and
=#f= edge conditions are emitted as JSON =null=.

It also recognizes a minimal workspace schema smoke form:

#+begin_src scheme
(workspace-schema workspace-record
  (required ID TITLE)
  (todo TODO DONE))
#+end_src

When =expected= is =WorkspaceSchema=, the adapter emits a =WorkspaceSchema=
JSON artifact with =schema_id=, =required_properties=, and =todo_states=.

The adapter now routes parser output through Gerbil-side modules before
emitting JSON:

- =fixtures/gerbil/marlin/protocol.ss= owns constructors/accessors for
  =LoopGraph=, =LoopNodeSpec=, =LoopEdgeSpec=, =WorkspaceSchema=, and
  =WorkspacePatchIntent=, and =AgentScenarioContract= plus the
  =GerbilCompileResponse= serializer and the supported artifact-kind registry.
- =fixtures/gerbil/marlin/request.ss= owns the =GerbilCompileRequest= JSON
  reader boundary through =:std/text/json=.
- =fixtures/gerbil/marlin/parser.ss= owns the Gerbil reader-backed smoke DSL
  source-to-=LoopGraph=, source-to-=WorkspaceSchema=, and
  source-to-=WorkspacePatchIntent=, and source-to-=AgentScenarioContract=
  parsers.
- =fixtures/gerbil/marlin/adapter.ss= imports the request/parser/protocol
  modules and owns the artifact compiler table, table/registry self-check, and
  response emission.
- =fixtures/gerbil/command-adapter.ss= is a thin launcher that imports
  =:marlin/adapter= and calls =run-marlin-command-adapter=.

This validates the real =gxi= process boundary and the first Gerbil-side
source-to-IR step before the compiler grows a full package/build boundary.

** WorkspacePatchIntent Smoke

The runtime smoke includes =WorkspacePatchIntent= as a JSON artifact that can
enter the Rust dry-run workflow. The source form compiles through
=protocol.ss=, =parser.ss=, and =adapter.ss= into =WorkspacePatchIntentSpec=
with =dry_run_first= set to true. The fixture source covers =set-todo=,
=set-property=, and =mark-memory-candidate= patch ops. The smoke proves the
real =gxi= boundary returns the typed artifact through both the launcher file
and module entrypoint, then proves Rust can pass the decoded intent into
=GerbilWorkspacePatchIntentDryRunner= and receive a =WorkspacePatchReceipt=.

The dry-run smoke does not execute a workspace mutation. The dry-run receipt
records affected nodes, deterministic dry-run hashes, and rejected memory
dispatch requests. Durable persistence is covered by a separate Rust workflow
request that pairs the decoded intent with an explicit document and
=OrgSourceWritePolicy=. The
=command_compiler_real_gxi_workspace_patch_intent_dry_runs_through_workflow=
test feeds the real =gxi= artifact into
=GerbilWorkspacePatchIntentDryRunner= and verifies the deterministic dry-run
receipt. The
=command_compiler_real_gxi_workspace_patch_intent_commits_with_policy= test
then compiles a real Gerbil =WorkspacePatchIntent=, pairs it with
=OrgSourceWritePolicy::write=, commits it through
=OrgWorkspaceSourceCommitter=, and verifies the resulting =memory.org= TODO
state and =OWNER= property.

** AgentScenarioContract Smoke

The runtime smoke includes =AgentScenarioContract= as a typed artifact emitted
by real =gxi=. The source form compiles through =protocol.ss=, =parser.ss=,
and =adapter.ss= into =AgentScenarioContract= with =schema_id=
=marlin.agent.scenario.v1=, one scenario step, expected event topics, expected
span names, and expected evidence. The Gerbil crate smoke proves artifact
production and decoding.

The harness crate owns the execution proof. The
=harness_executes_real_gxi_loop_graph_against_real_gxi_scenario_contract=
ignored test compiles both a Gerbil =LoopGraph= and a Gerbil
=AgentScenarioContract=, projects the graph into the protocol =LoopGraph=,
executes it through =HarnessRuntime= and =TokioGraphLoopKernel=, and evaluates
the resulting =HarnessExecutionReport= through
=AgentHarness::evaluate_contract_execution_report=. That is the first
Scheme-produced artifact path wired into graph-loop execution.

** Real =gxi= Gate

The explicit local Gerbil gate is:

#+begin_src sh
cargo run -p marlin-gerbil-scheme --bin marlin-gerbil-real-gxi-gate
#+end_src

For CI logs and local diagnosis, print the typed gate plan without executing
tests:

#+begin_src sh
cargo run -p marlin-gerbil-scheme --bin marlin-gerbil-real-gxi-gate -- --print-plan
#+end_src

The plan is newline-delimited key-value text with =workspace_root=, =cargo=,
=packages=, =test_binary=, =filter=, =ignored=, =gerbil_home=, =gxi=, and
=command=. It is intentionally not a Scheme DSL and not part of the native C
ABI path.

The gate validates the local =gxi= executable and Gerbil home before invoking:

#+begin_src sh
cargo test -p marlin-gerbil-scheme -p marlin-agent-harness --locked --test unit_test real_gxi -- --ignored
#+end_src

Use =--filter= to narrow the gate to a specific package-local test path when
debugging, but the default gate must remain cross-crate so Scheme artifact
production and harness loop execution stay connected.

** Launcher

The launcher requires =GERBIL_LOADPATH= to point at the fixture root so the
=package: marlin= modules resolve:

#+begin_src sh
GERBIL_LOADPATH=crates/marlin-gerbil-scheme/fixtures/gerbil \
  /opt/homebrew/opt/gerbil-scheme/bin/gxi \
  crates/marlin-gerbil-scheme/fixtures/gerbil/command-adapter.ss
#+end_src

The same adapter module can be invoked directly through Gerbil's module
entrypoint without the launcher file:

#+begin_src sh
GERBIL_LOADPATH=crates/marlin-gerbil-scheme/fixtures/gerbil \
  /opt/homebrew/opt/gerbil-scheme/bin/gxi \
  :marlin/adapter
#+end_src

The Rust boundary tests pass that value through =GerbilCommandSpec.env=, not
through a shell wrapper. They cover the source-file launcher and module entry,
successful =LoopGraph=, =WorkspaceSchema=, =WorkspacePatchIntent=, and
=AgentScenarioContract= compilation, the =WorkspacePatchIntent= dry-run and
policy-gated commit workflow receipts, plus Gerbil-side rejection of
unsupported =expected= artifact kinds and mismatched source forms.

** Runtime Assets And Example Scheme

=marlin-gerbil-scheme= ships the Gerbil binding sources as crate-owned runtime
assets. =write_gerbil_runtime_assets= writes =command-adapter.ss=, =smoke.ss=,
and the =marlin/= modules into a caller-provided loadpath root. A local runtime
can then run the module entrypoint with =GERBIL_LOADPATH=<root> gxi
:marlin/adapter=, or run the example source directly. Rust callers can use
=GerbilCommandCompiler::from_marlin_runtime_module= for the same module
entrypoint, or =GerbilCommandSpec::marlin_runtime_launcher= for the source-file
launcher. Local smoke tests discover =gxi= through =MARLIN_GERBIL_GXI= before
falling back to the Homebrew =gerbil-scheme= path.

=examples/workspace-patch-intent.ss= is the minimal user-facing Gerbil scheme
example. It imports =:marlin/parser= and =:marlin/protocol=, compiles a
=WorkspacePatchIntent= source form containing =set-todo=, =set-property=, and
=mark-memory-candidate=, and emits the same JSON =GerbilCompileResponse= shape
that Rust decodes from the command adapter. The
=command_compiler_real_gxi_runs_workspace_patch_intent_example_from_runtime_assets=
smoke writes the crate assets to a temporary loadpath, runs that example through
real =gxi=, decodes the JSON response in Rust, and verifies the typed
=WorkspacePatchIntent= artifact.

=examples/workspace-patch-intent.rs= is the Rust-facing example. It writes the
crate-owned Gerbil runtime assets to a temporary loadpath, constructs a
=GerbilCommandCompiler= with =from_marlin_runtime_module=, compiles a
=WorkspacePatchIntent= source form through real =gxi=, and prints the intent id,
=dry_run_first= gate, and patch operation count.

#+begin_src sh
cargo run -p marlin-gerbil-scheme --example workspace-patch-intent
#+end_src

=examples/workspace-patch-intent-source.ss= is the source-file form consumed by
the workflow example. =examples/workspace-patch-intent-workflow.rs= reads that
=.ss= file, compiles it through the crate-owned runtime binding, feeds the
decoded =WorkspacePatchIntent= into =GerbilWorkspacePatchIntentDryRunner=, and
prints the workflow receipt/evidence summary.

#+begin_src sh
cargo run -p marlin-gerbil-scheme --example workspace-patch-intent-workflow
#+end_src

The package audit must include both examples and the Gerbil runtime assets:

#+begin_src sh
cargo package -p marlin-gerbil-scheme --list --allow-dirty
#+end_src

Run the Rust boundary tests explicitly:

#+begin_src sh
direnv exec . cargo test -p marlin-gerbil-scheme command::real_gxi -- --ignored --nocapture
#+end_src

The tests are ignored by default because they require a local Gerbil installation.
