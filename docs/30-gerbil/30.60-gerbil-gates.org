* 30.60 Gerbil Gates

** Default CI Gate

The default gate must not require a local Gerbil installation. It proves the
Rust API surface, package contents, examples, and workflow handoff while leaving
real =gxi= execution behind ignored tests.

#+begin_src sh
direnv exec . cargo fmt --all -- --check
direnv exec . cargo test --workspace
direnv exec . cargo clippy --workspace --all-targets --all-features -- -D warnings
asp rust check --changed .
#+end_src

Expected outcome:

- =marlin-gerbil-scheme= unit tests pass with real =gxi= tests ignored.
- =marlin-org-workflow= proves dry-run, contract-fact rejection,
  policy-gated commit, dry-run source policy, and missing-document rejection.
- Example binaries type-check under =cargo clippy --all-targets=.
- Harness build gates keep large tests split into folder-first suites.

** Local Real-GXI Gate

Run this gate on machines with Gerbil installed. The crate discovers =gxi=
through =MARLIN_GERBIL_GXI= first, then falls back to the Homebrew
=gerbil-scheme= path.

The workflow calls the native gate binary instead of duplicating the cargo test
filter inline. Print the plan first when auditing a runner:

#+begin_src sh
cargo run --locked -p marlin-gerbil-scheme --bin marlin-gerbil-real-gxi-gate -- --print-plan
cargo run --locked -p marlin-gerbil-scheme --bin marlin-gerbil-real-gxi-gate
#+end_src

#+begin_src sh
direnv exec . cargo test -p marlin-gerbil-scheme --test unit_test command::real_gxi -- --ignored --nocapture
direnv exec . cargo run -p marlin-gerbil-scheme --example workspace-patch-intent
direnv exec . cargo run -p marlin-gerbil-scheme --example workspace-patch-intent-workflow
#+end_src

Expected outcome:

- The ignored =command::real_gxi= suite passes artifact, error, example,
  dry-run workflow, and policy-gated commit smoke tests.
- =workspace-patch-intent= prints =compiled intent:memory=, =dry_run_first true=,
  and the emitted patch operation count.
- =workspace-patch-intent-workflow= reads
  =examples/workspace-patch-intent-source.ss=, compiles through the crate-owned
  =:marlin/adapter= runtime, dry-runs the intent, and prints
  =workflow_accepted true= plus =evidence_present true=.

If the gate fails after a Gerbil/Gambit version repair, first verify the user
package surface. Stale =clan= artefacts can report old/new compiler mismatches
or missing exports, and a missing =clan/poo= build reports
=cannot find library module :clan/poo/object=.

#+begin_src sh
env GERBIL_HOME="$PWD/.data/gerbil/build" \
  .data/gerbil/build/bin/gxi -e '(begin (import :clan/base) (display "clan-base-ok") (newline))'
env GERBIL_HOME="$PWD/.data/gerbil/build" \
  .data/gerbil/build/bin/gxi -e '(begin (import :clan/poo/object) (display "poo-object-ok") (newline))'
rm -rf "$HOME/.gerbil/lib/clan"
env GERBIL_HOME="$PWD/.data/gerbil/build" \
  .data/gerbil/build/bin/gxpkg build clan
env GERBIL_HOME="$PWD/.data/gerbil/build" \
  .data/gerbil/build/bin/gxpkg build --dependencies clan/poo
#+end_src

Observed local evidence on 2026-06-11:

#+begin_src text

test result: ok. 17 passed; 0 failed; 0 ignored; 26 filtered out
#+end_src

** CI Gate Matrix

[[file:../../.github/workflows/ci.yaml][.github/workflows/ci.yaml]] keeps the
default Rust gate separate from gates that require a real Gerbil/Gambit native
toolchain.

| Gate | Trigger | Real tool requirement | Command boundary |
|-
| Rust format/check/lint/test | push, pull request, manual | none | =cargo fmt=, =cargo check --workspace --all-targets=, =cargo clippy --workspace --all-targets=, =cargo test --workspace= |
| Gerbil package surface | push, pull request, manual | none | =cargo test -p marlin-gerbil-ir -p marlin-gerbil-scheme -p marlin-org-workflow=, binary/example check, package asset audit |
| Real =gxi= release gate | manual =run_real_gxi= | executable =gxi= via =MARLIN_GERBIL_GXI= or Homebrew =gerbil-scheme= | =marlin-gerbil-real-gxi-gate --print-plan=, then =marlin-gerbil-real-gxi-gate= |
| Linked native AOT gate | manual =run_linked_native= | executable =gsc= and =gxc= via =MARLIN_GERBIL_GSC= / =MARLIN_GERBIL_GXC= or Homebrew =gerbil-scheme=; =clang= as C compiler | =cargo test -p marlin-deck-runtime-native --features linked-native= and =cargo bench -p marlin-deck-runtime-native --features linked-native --bench linked_selector= |

The manual gate runners are configurable through JSON =runs-on= inputs. Use
=real_gxi_runner= or =linked_native_runner= for self-hosted Linux or macOS
runners that already provide Gerbil/Gambit. Generic hosted Linux runners are
not treated as Gerbil-ready unless the required executables are supplied through
repository variables or the runner image.

The linked native AOT runner infers the Gerbil/Gambit prefix from =gsc= and
=gxc=, then injects =GERBIL_HOME= and =GAMBOPT= for compiler subprocesses. CI
therefore needs real tool paths, not a pre-created =~~bin= repair directory.

** Distribution Boundary

The crate is currently an internal workspace crate, not an independently
published crates.io package. Its manifest uses =publish = false= while the
workspace dependency graph is still path-owned.

#+begin_src sh
direnv exec . cargo package -p marlin-gerbil-scheme --list --allow-dirty
#+end_src

This gate audits the distributable crate contents without asking Cargo to
resolve unpublished workspace-internal crates from the registry.

Expected contents include:

- =README.md=
- =examples/workspace-patch-intent.rs=
- =examples/workspace-patch-intent-workflow.rs=
- =examples/workspace-patch-intent-source.ss=
- =fixtures/gerbil/command-adapter.ss=

- =fixtures/gerbil/command-adapter-batch.ss=
- =fixtures/gerbil/marlin/adapter.ss=
- =fixtures/gerbil/marlin/parser.ss=
- =fixtures/gerbil/marlin/protocol.ss=
- =fixtures/gerbil/marlin/request.ss=

The real-gxi CI gate also writes =release-status.json= and
=release-landing-report.json= into =MARLIN_RELEASE_STATUS_ARTIFACT_DIR= so a
failed or passing local-Gerbil release run leaves a file-backed status receipt.

External publishing is a separate release task. Before enabling it, publish or
otherwise provide the internal dependency chain first, including
=marlin-agent-protocol=, =marlin-gerbil-ir=, =marlin-org-model=, and the
Gerbil/workspace protocol crates that =marlin-gerbil-scheme= depends on. The
release topology is tracked in
[[file:30.70-gerbil-release-topology.org][30.70 Gerbil Release Topology]].
