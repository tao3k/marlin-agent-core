* 30.50 Gerbil Bindings Reference

This note records the first external binding reference used for
=marlin-gerbil-scheme=.

** Source Baseline

- Repository: =https://git.cons.io/mighty-gerbils/gerbil-mysql=
- Local checkout: =.data/gerbil-mysql=
- Audited commit: =847b4b61747152e505dfc72acee4438009302c5f=

** Binding Shape

=gerbil-mysql= uses a three-layer binding shape:

- =db/_mysql.scm= owns the low-level Gambit FFI surface with =c-declare=,
  =c-define-type=, =c-lambda=, and helper macros such as =define-c-lambda=.
- =db/_mysql.ssi= owns the Gerbil interface boundary with =package:=,
  =export=, and =extern= declarations for the low-level functions.
- =db/mysql.ss= owns the public Gerbil API. It imports the low-level module and
  wraps pointer/error/async behavior behind Gerbil-level functions and structs.

The build script compiles the low-level FFI module first, builds the =.ssi=
interface, and then builds the public module:

#+begin_src scheme
(gsc: "db/_mysql" ...)
(ssi: "db/_mysql")
"db/mysql"
#+end_src

** Marlin Application

For Marlin, the same pattern maps to:

- a low-level native layer later, if we embed or link against Gerbil/Gambit
  directly;
- a Gerbil protocol binding layer now, where Gerbil owns constructors,
  accessors, and typed Marlin protocol values;
- a Gerbil request binding layer now, where Gerbil exposes typed
  =GerbilCompileRequest= accessors over Rust-owned projections;
- a Gerbil reader-backed parser layer now, where Gerbil compiles source text
  into protocol values;
- an adapter layer that dispatches typed requests, calls the parser, and returns
  typed artifacts for Rust projection.

The current smoke implementation therefore keeps native VM bindings out of the
Rust crate while still moving bindings out of ad hoc text or JSON protocol
construction.
The first binding module is:

#+begin_src text
crates/marlin-gerbil-scheme/gerbil/src/marlin/protocol.ss
#+end_src

The first parser module is:

#+begin_src text
crates/marlin-gerbil-scheme/gerbil/src/marlin/parser.ss
#+end_src

The first request module is:

#+begin_src text
crates/marlin-gerbil-scheme/gerbil/src/marlin/request.ss
#+end_src

The first adapter module is:

#+begin_src text
crates/marlin-gerbil-scheme/gerbil/src/marlin/adapter.ss
#+end_src

These modules use =package: marlin= and resolve through =GERBIL_LOADPATH= during
the smoke test.

** Next Work

- Add a Gerbil build/install script for the crate-shipped runtime modules once
  the package boundary stabilizes.
- Only introduce native =.scm/.ssi= FFI bindings after the command protocol and
  typed IR have stabilized.
- Keep the default CI gate free of local =gxi= requirements. The real runtime
  gate is documented separately in
  [[file:30.60-gerbil-gates.org][30.60 Gerbil Gates]] so binding work can move
  forward without making ordinary workspace checks depend on a machine-local
  Gerbil installation.
