* 10.20 Kernel Boundary
:PROPERTIES:
:ID: docs-10-20-kernel-boundary
:KIND: architecture
:STATUS: active
:END:

The kernel owns graph execution authority, runtime snapshots, provider/tool
runtime boundaries, sub-agent runtime boundaries, events, cancellation, and the
native extension registry.

The kernel executes provider-neutral =LoopProgram= artifacts through a generic
loop machine. It does not own provider-branded turn drivers. Codex-style,
Claude-style, OpenRath-style, ReAct, maker-checker, and planner-executor
behaviors enter the system as POO Flow loop mechanism policies that compile
into typed transitions and receipts.

The kernel projects loop actions into typed runtime handoff plans and typed
handoff execution receipts. Runtime adapters own the concrete handlers for
model, tool, memory, placement, session, agent, verification, human-gate,
receipt, and graph lanes.

Provider adapters own model protocol adaptation only: =ModelRequest= enters the
provider, and =ModelEvent=, =ToolRequest=, or =FinalResponse= leaves it. They do
not own loop semantics, graph rewrite, pending-input drain, retry, approval, or
verification policy.

The kernel consumes workspace and Gerbil IR contracts. It does not own Org
parsing, memory algorithms, external storage, or transport systems.
