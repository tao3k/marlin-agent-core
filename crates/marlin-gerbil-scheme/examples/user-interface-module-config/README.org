* User Interface Workspace Example
This directory is the downstream user-facing example for Marlin policy handoff through the poo-flow user-interface style.

The user path mirrors =poo-flow/user-interface=: a tiny =init.ss= selects module categories, and the custom module body is split into small =use-module= fragments. Marlin keeps the furnished policy pack, delivery receipt, option contracts, projection chain, loop governor handoff, and Rust runtime boundary upstream.

** Layout
- =init.ss= is the user entrypoint. It is the Marlin equivalent of poo-flow's Doom-style =user-interface/init.ss=.
- =custom/marlin-user-interface/config.ss= is the custom module body. It only calls =load!= for profile and case fragments.
- =custom/marlin-user-interface/profiles/*.ss= and =custom/marlin-user-interface/cases/*.ss= are downstream declarations. They use poo-flow =use-module= selections and POO =.def= objects where the upstream module family supports native config.
- =../../gerbil/src/config-interface/modules/prefabs/user-interface.ss= is the upstream furnished policy surface. It owns default imports, extensions, script wiring, continuation profile, subagent policy, and policy-pack object surgery.
- =../../gerbil/src/config-interface/modules/prefabs/user-interface-delivery.ss= is the upstream delivery surface. It owns the delivery receipt, option contract projection, poo-flow loop governor manifest, and Rust policy projection handoff.
- =t/user-interface-module-config-test.ss= runs the downstream selections and Marlin adapter with real Gerbil.
- =interface.org= and =state/worker-state.org= are workspace files consumed by the prefab script.

** User Interface
The public surface is the poo-flow user-interface style: users select module categories and keep custom behavior in small fragments.

#+begin_src scheme
(import :poo-flow/src/module-system/root-profile)

(poo-flow!
 :workflow
 (funflow)
 (loop-engine)
 :sandbox
 (nono-sandbox +nono +doctor)
 :custom
 (marlin-user-interface "./custom/marlin-user-interface" +private +doctor))
#+end_src

The custom module fragments stay declarative. This example is not installed as a Gerbil package, so its =config.ss= uses module-root-qualified =load!= paths when run directly from the examples directory. The adapter test handles the generated fallback binding names, projects =init.ss= through poo-flow's root-profile lib, and passes the POPflow module bundle to Marlin's furnished =UserInterfaceWorkspaceFromModuleBundle= bridge; the user still edits small fragments with =use-module=, matching poo-flow's =custom/my-module= shape.

The loop smoke follows poo-flow's loop profile shape: the downstream
=loop-engine= object declares =+manifest-handoff= and =+l1-receipts=, and the
=funflow= pipeline keeps =runtime-executed= as =#f=. This keeps the default test
as a no-live-LLM handoff proof before a separately gated real-LLM exercise is
enabled.

The live exercise is the stream gateway gate, not the default smoke:

#+begin_src sh
MARLIN_LIVE_LLM_GATE=1 \
MARLIN_LIVE_LLM_PROVIDER="$PROVIDER" \
MARLIN_LIVE_LLM_MODEL="$MODEL" \
cargo test -p marlin-agent-stream --locked \
  live_litellm_stream_gateway_completes_provider_neutral_request \
  -- --ignored --nocapture
#+end_src

Provider credentials stay outside the module config. Use the provider's normal
API-key environment variable, or set =MARLIN_LIVE_LLM_PROVIDER_API_KEY_ENV= to
the variable name that should be checked.

For example, with =DEEPSEEK_API_KEY= available, use =PROVIDER=deepseek= and
=MODEL=deepseek-chat=.

Generic Marlin workspace values are symbol-keyed flags:

#+begin_src scheme
(use-module marlin-user-interface
  (workspace-root . "user-interface-workspace")
  (interface-file . "interface.org")
  (state-file . "state/worker-state.org")
  (model-profile . "interactive"))
#+end_src

Loop, sandbox, and verification declarations use upstream poo-flow module families:

#+begin_src scheme
(use-module loop-engine
  :config
  (.def (marlin-ui-loop @ loop-engine-use-case name level mode goal)
    name: 'marlin-user-interface
    level: 'l1
    mode: 'report-only
    goal: 'produce-policy-delivery-receipt))
#+end_src

Users do not assemble raw poo-flow descriptors, import graphs, extension catalogs, policy-pack object lists, runtime projection helpers, loop-governor constructors, or option contract receipts in this layer. The test/adapter layer reads the selections and then calls the Marlin furnished prefab to produce the delivery receipt.

** Run
From this directory:

#+begin_src sh
GERBIL_LOADPATH=../../gerbil/src:../../gerbil/.gerbil/lib:.:t /usr/local/bin/gxi t/user-interface-module-config-test.ss
#+end_src

The minimal user entrypoint can also be loaded directly:

#+begin_src sh
GERBIL_LOADPATH=../../gerbil/src:../../gerbil/.gerbil/lib:. /usr/local/bin/gxi init.ss
#+end_src

The Rust ignored real-Gerbil gate copies and runs this same directory, so the example and the executable downstream test stay aligned.
