* User Interface Module Config Example

This directory is the downstream user-facing example for the furnished Gerbil
POO module system. The default path is intentionally small: users import the
upstream prefab, provide a tiny POO config object, and run the workflow.

** Layout

- =user-interface-modules/config.ss= is the user entrypoint. It only chooses
workspace files and a model profile.
- =../../gerbil/src/modules/prefabs/user-interface.ss= is the upstream
furnished policy surface. It owns default imports, extensions, script wiring,
continuation profile, subagent policy, and policy-pack object surgery.
- =../../gerbil/src/modules/prefabs/user-interface-delivery.ss= is the upstream
delivery surface. It owns the user-facing receipt, option contract projection,
and Rust policy projection handoff.
- =../../gerbil/src/modules/prefabs/default-policy.ss= is the shared default
policy pack surface inherited by furnished prefabs.
- =t/user-interface-module-config-test.ss= runs the workflow with =gxtest=.
- =interface.org= and =state/worker-state.org= are workspace files consumed by
the prefab script.

** User Interface

The public surface is the furnished-house API from the design document: import
a prefab and provide only config values.

#+begin_src scheme
(import :clan/poo/object
        :modules/prefabs/user-interface-delivery)

(def user-interface-module-config
  (UserInterfaceWorkspace
   (.o workspace-root: "user-interface-module-config"
       interface-file: "interface.org"
       state-file: "state/worker-state.org"
       model-profile: "interactive")))
#+end_src

Users do not assemble =marlinModules=, import graphs, extension catalogs,
policy-pack object lists, runtime projection helpers, or option contract
receipts in this layer. The prefab already contains the normal workspace,
agent, hook, continuation, subagent, script, memory trigger, catalog projection
defaults, and a single delivery receipt for debug/Rust handoff.

The example keeps only the handoff helpers a downstream user should expose:

#+begin_src scheme
(def (user-interface-delivery-receipt)
  (UserInterfaceDeliveryReceipt user-interface-module-config))
(def (user-interface-policy-apply)
  (UserInterfaceApply user-interface-module-config))

(def (user-interface-policy-projection)
  (UserInterfacePolicyProjection user-interface-module-config))
#+end_src

** Customization

Customization should be object-level surgery against the furnished prefab, not
manual module-system construction. Advanced callers can replace or add typed
policy objects with the =marlin-add-object=, =marlin-remove-object=,
=marlin-disable-object=, and =marlin-replace-object= helpers exported by
=:modules/lib=.

The furnished user-interface pack inherits the upstream default policy pack and
then applies object surgery for the user-interface-specific subagent,
continuation, hook, and memory policy objects. The low-level =marlinModules=,
=marlinModuleCatalog=, =UserInterfaceWorkspaceWorkflow=, and
=UserInterfacePolicyPack= APIs still exist for framework authors and tests, but
they are not the default user interface.

** Run

From this directory:

#+begin_src sh
GERBIL_LOADPATH=../../gerbil/src:user-interface-modules:t gxtest -v t/user-interface-module-config-test.ss
#+end_src

The Rust ignored real-gxtest gate copies and runs this same directory, so the
example and the executable downstream test stay aligned.
